\section{Introduction}

Finite-context world models often fail in a way that is easy to miss: they can
observe the decisive cue early and later produce a representation that behaves
as if the cue was never present.  This matters for physical intelligence
because action-relevant evidence is rarely synchronized with the moment of
action.  A robot can read a sign before turning away, see a door state before
entering a hallway, or observe an object identity before the object becomes
occluded.  If the interface to the frozen host model exposes only a recent
window, then locally plausible prediction is not enough; the old evidence must
survive long enough to affect the later decision.

This paper studies that failure as an interface problem.  We do not ask only
whether a hidden state can memorize something.  The stricter question is
whether old visual evidence, after it has left the legal context window, is
retained in memory, exposed through a host-facing representation, and usable by
a bounded downstream controller.  This framing separates three claims that are
often conflated.  A model may store old evidence internally but fail to expose
it.  A readout may decode a label from a shortcut if the current endpoint still
leaks the cue.  A native controller may succeed because it received the true
goal rather than the memory-selected one.  Our evaluation is designed to close
these loopholes.

We introduce a masked-evidence slot-memory interface.  Training is
self-supervised: it never receives cue labels, cue crops, or manually selected
key frames.  Instead, the training pipeline mines visually salient patch sets
from the rendered causal stream, masks those patches from the visible source,
and asks a compact slot transformer to predict their stop-gradient latent
targets.  Cue labels enter only after training, when a post-hoc audit readout
tests whether the learned memory exposes the old cue under matched full,
reset, and no-state conditions.

The central empirical point is structural rather than just numerical.  Under
the same caches, target mining, patch target encoder, loss, optimizer, and
readout, replacing the slot writer with GRU, LSTM, or Mamba-lite recurrence
weakens long-delay exposure.  The baselines can often solve short waypoint-like
rows, which is useful evidence that the task is not impossible.  Their failure
appears when the cue is old or distributed across object-like visual evidence.
Slots help because the target itself is a set of visual facts; a memory set is
a better carrier for that target than a single hidden vector.

Our contributions are: a fail-closed audit for old-evidence memory in frozen
world-model interfaces; a label-free masked patch-set objective for training a
slot memory writer; a controlled architecture comparison across twelve OGBench
render decks, three evidence ages, and three seeds; and a bounded native-use
check showing that the exposed memory changes target selection where audited
fixed controllers are available.

\section{Related Work}

\paragraph{World models and finite context.}
Latent world models compress observations into predictive state and use that
state for control or planning \citep{r_dreamer}.  This compression is powerful
but makes old evidence a first-class interface concern: the host can produce
reasonable short-horizon predictions while dropping information needed for a
later choice.  OGBench supplies diverse offline goal-conditioned environments
\citep{r_ogbench}; we use rendered OGBench decks as a controlled substrate for
old-evidence tests rather than as an end-to-end planner benchmark.

\paragraph{Self-supervised prediction.}
JEPA-style objectives predict hidden representations instead of pixels
\citep{r_jepa}, and contrastive objectives such as InfoNCE encourage a model
to distinguish the positive target from alternatives \citep{r_cpc}.  Our use
case is narrower: the target is not the next frame, but visual evidence that
has been deliberately removed from the legal context.  The target miner is
generic and label-free, so the audit can later ask whether semantic cue
information emerged without being trained as a class label.

\paragraph{Memory architectures.}
GRU and LSTM recurrence \citep{r_gru,r_lstm} provide classic sequence memory,
and selective state-space models such as Mamba \citep{r_mamba} provide a
stronger long-sequence prior.  Transformer attention \citep{r_transformer} and
slot-based object representations \citep{r_slot_attention} motivate a
different view: memory should be a structured set when the evidence itself is
multi-part.  Our experiments keep the objective fixed and vary the writer, so
the comparison probes the memory interface rather than the supervision signal.

\section{Preliminaries}

Let $x_{0:T}$ denote rendered frames and $a_{0:T-1}$ actions.  A cue $c$ is
visible at an early time and determines a later target label $y$.  At readout
time $t$, the frozen host sees only a recent context window of length $K$; the
cue is outside that window.  We call the delay between cue visibility and the
readout boundary the evidence age.  The current registered ages are 4, 8, and
15, which act as controlled stress tests rather than automatically discovered
memory horizons.

The audit has three conditions.  The \emph{full} condition uses the causal
memory state $M_t$.  The \emph{reset} condition observes the same endpoint and
recent context but clears persistent memory at the context boundary.  The
\emph{no-state} condition removes persistent memory entirely.  A cell passes
only when full-memory balanced accuracy is high while both controls remain
near chance.  This fail-closed rule is the important part: a high full score is
not accepted if the endpoint or recent context can solve the task.

We additionally run a bounded native-use check where a local audited
controller exists.  The memory readout selects a target; the controller receives
that selected target rather than the true label; the environment reports
success against the original target.  This is not autonomous planning.  It is
a controlled test that exposed memory changes a downstream physical outcome.

successes.  They are marked as controller-unavailable and remain outside the
native-use claim.

\section{Methodology}

Figure~\ref{fig:architecture} summarizes the method as the paper uses it.  The
left side is the physical stream: frames, actions, and time indices arrive
causally.  The target side is built from the same stream but is hidden from the
visible source.  The middle is the learned memory writer.  The right side is
the audit, which is intentionally outside the training loss.

\centering
\footnotesize
\caption{Module contract.  The training path is label-free; labels appear only
in the post-training audit path.}
\label{tab:module-contract}
\begin{tabularx}{\linewidth}{lYY}
\toprule
Module & Input and output & Design reason\\
\midrule
Trajectory cache
  & Rendered frames, actions, time indices, and split metadata.
  & Keeps all methods on the same stream and validation rows.\\
Patch miner
  & Selects salient target patches from the causal history without labels.
  & Prevents a manual cue crop from becoming the supervision signal.\\
Slot writer
  & Maps visible tokens into a small persistent slot set $M_t$.
  & Gives distributed old evidence multiple carriers instead of one vector.\\
Set predictor
  & Predicts unordered target-patch latents with InfoNCE and cosine matching.
  & Matches the multi-patch target while avoiding a hand-coded slot order.\\
Audit readout
  & Fits labels after training and evaluates full/reset/no-state features.
  & Measures exposed old evidence rather than the self-supervised loss alone.\\
\bottomrule
\end{tabularx}
\end{table}

The module contract clarifies why the comparison to recurrent baselines is
meaningful.  GRU, LSTM, and Mamba-lite are not trained on weaker data.  They
receive the same mined targets and are judged by the same readout gate.  The
difference is the memory format used to carry old evidence from the causal
history to the host-facing representation.

\paragraph{Masked patch-set target.}
For each episode, the target miner divides the causal history into temporal
bins and selects visually salient, novel patches from those bins.  The miner
does not know cue labels, cue positions, or task-specific object crops.  The
selected patches are removed from the visible stream and encoded by a fixed
target encoder into color and texture latents.  This design makes the training
signal old and visual without making it semantic supervision.

\paragraph{Slot writer.}
The writer receives frame tokens, action tokens, and time tokens.  A compact
slot transformer updates a fixed number of normalized memory slots.  Prediction
is set-valued: an MLP maps slots to target-patch latents, and the loss combines
set InfoNCE, cosine matching, and a small anti-collapse standard-deviation
regularizer.  Because both the memory and target are sets, the model is not
forced to squeeze every old fact into one recurrent vector or one hand-indexed
slot.

\paragraph{Shared audit readout.}
After training, a ridge readout is fitted on train memory features and cue
labels.  The same readout is then applied to held-out full, reset, and no-state
features.  Labels are therefore read only for evaluation, not for optimizing
the memory writer.  The baselines replace the slot writer with GRU, LSTM, or a
plain PyTorch Mamba-lite recurrence, but keep the target miner, target encoder,
loss, features, and readout contract unchanged.

The training and inference procedures are deliberately asymmetric.  During
training, the model only receives reconstruction pressure on hidden patch sets.
During evaluation, the readout asks a semantic question that was not optimized
directly.  This asymmetry is important: if labels were used during training, a
reviewer could attribute success to supervised cue classification.  If labels
were never used at all, there would be no way to audit whether the memory
retained the task-relevant cue.  The split between label-free training and
label-revealed evaluation is therefore a measurement design, not a loophole.

\section{Experimental Results}

\paragraph{Protocol.}
We evaluate twelve rendered OGBench decks: PointMaze medium, large, giant, and
teleport; AntMaze large and giant; HumanoidMaze large; Cube single, double, and
triple; Puzzle-3x3; and Scene.  Every memory writer is run at the same three
evidence ages and three seeds.  The result artifacts used by this paper are
generated from local JSON and NPZ outputs by \path{scripts/plot_paper_c.py}.

\input{generated_results/main_comparison.tex}

Table~\ref{tab:main-comparison} is the compact scorecard, but the interpretation
is not that a single average proves memory.  The full/reset/no-state pattern is
what matters.  The slot model exposes old evidence while the matched controls
stay close to chance, so the readout is not merely exploiting endpoint color,
action sequence, or short-window context.  The recurrent baselines are useful
negative controls: they share the same self-supervised target and can solve a
large fraction of easier cells, yet they lose the consistent exposure pattern
on older and more structured evidence.

Two details make this comparison stronger than a raw leaderboard.  First, the
controls are matched at the feature level: full, reset, and no-state readouts
are evaluated on the same held-out episodes.  A method cannot pass by making
every endpoint easy, because the controls would pass as well.  Second, the
baselines are strong enough to reveal where the task changes character.  Their
short-age successes show that the target miner is learnable and that the
readout is not artificially hostile.  Their long-age failures show that the
memory interface, not only the loss, is doing work.

Figure~\ref{fig:cue-inference} shows the mechanism behind the scorecard.  The
full condition does not merely increase an aggregate metric; it moves class
evidence toward the old cue on individual held-out episodes.  Reset and
no-state do not become uniformly random on every example.  Instead, they
collapse toward biased columns that are plausible from the endpoint but wrong
for many true cues.  This is why the controls are essential: they show what the
frozen endpoint can guess without old evidence.

This plot is also a useful diagnostic for future failures.  If full, reset, and
no-state all concentrate on the true cue, the deck leaks the answer and should
be rejected.  If all three conditions concentrate on the same wrong column, the
readout is dominated by endpoint bias.  If full is correct but reset has a high
patch-retrieval score, the model may be reconstructing generic visual content
without exposing the semantic cue.  The desired pattern is the one shown here:
full memory aligns with the old cue while the controls reveal the best shortcut
available without memory.

The age sweep explains where the architecture matters.  Short-delay evidence is
not the hard regime: a recurrent state can often carry a simple cue for a few
steps.  The separation appears as the model must preserve several visual facts
through a longer irrelevant interval.  LSTM decays first, GRU and Mamba-lite
remain stronger but fall below the registered gate on many long-age rows, and
the slot writer keeps a stable gap.  This supports the design hypothesis that
set-structured memory is aligned with set-structured visual evidence.

The age curve should not be read as a claim that the chosen delays are the only
delays that matter.  They are probe points.  Age 4 asks whether evidence just
outside the context boundary remains accessible.  Age 8 tests whether the
memory is more than a short transient.  Age 15 is the current long-delay stress
test under the available caches.  The shape of the curve matters because it
shows graceful exposure under increasing delay rather than a single lucky age.

Figure~\ref{fig:gap} and Table~\ref{tab:hard-rows} are more informative than
the average alone.  The baselines remain close on several locomotion-like rows,
which argues against an unfairly weak comparison.  The larger gaps are in
Puzzle, Scene, Cube, and long-age PointMaze rows, where the relevant evidence
is distributed over object identity, order, or spatial configuration.  These
are the cases where a single hidden state has the most trouble exposing the old
cue in a linearly readable form.

The heatmap also prevents overclaiming.  Some navigation rows have small gaps,
which means the recurrent baselines are already close when the cue behaves like
a single waypoint.  The contribution is not that slots dominate every possible
sequence memory problem.  The contribution is that the slot format remains
stable when old evidence is compositional: several patches, several objects, or
several spatial alternatives must be kept distinct until the readout point.
That is the physical-intelligence regime this paper targets.

The native-use stage asks a different question from the readout audit.  It
does not prove autonomous planning, but it does test whether the exposed memory
can change a physical outcome under a fixed controller.  The key observation is
the large separation between full memory and recent-only or random targets.
That separation means the memory readout is not just decodable in a notebook;
it selects a target that the controller can execute.  AntMaze and HumanoidMaze
are intentionally excluded from this native-use table because the repository
does not yet contain an audited local low-level controller for those bodies.

The native-use result is best understood as a bridge between representation and
action.  The controller is intentionally simple and fixed, so it cannot repair
a wrong memory target.  When recent-only and random targets fail but the
full-memory target succeeds, the most direct explanation is that old evidence
changed the goal passed into the controller.  This is weaker than learning a
policy with memory end to end, but stronger than reporting a readout table
alone.

\paragraph{Discussion.}
The experimental picture is coherent.  The target miner creates an old,
label-free visual prediction problem.  The slot writer provides a memory format
that matches the target's multi-part structure.  The readout audit shows that
old evidence is exposed rather than hidden inside an inaccessible state.  The
fixed-controller stage shows that exposed evidence can alter execution when a
valid controller exists.  The strongest claim is therefore not ``the model
solves memory in general.''  It is more precise: under controlled evidence
ages, a masked-evidence slot interface retains and exposes old visual evidence
more reliably than recurrent alternatives trained with the same objective.

This framing also explains what the current evidence does not yet establish.
The ages are registered stress tests, not learned retrieval horizons.  The
memory target is mined automatically, but the evaluation still chooses specific
age values.  A separate auto-age extension should train on sampled delays and
evaluate unseen delays with a query router over timestamped slots.  That
extension would answer a different question: whether the model can decide how
far back to retrieve, not merely whether it can retain old evidence at known
ages.

For ICLR 2027 positioning, the cleanest paper claim is therefore a controlled
memory-interface result.  The evidence is adequate for that claim because it
includes demand construction, label-free training, matched controls, recurrent
architecture baselines, environment-level visual analysis, and bounded native
use.  It is not yet adequate for a broader claim about autonomous memory
management.  That broader claim should wait for the auto-age router and
unseen-delay evaluation.

\section{Conclusion}

Old-evidence memory should be evaluated by retention, exposure, and bounded
use.  A hidden state that stores information but never exposes it is not enough
for a frozen world-model interface; a readout that succeeds without controls is
not enough either.  The masked-evidence slot-memory interface satisfies this
stricter ladder on the registered controlled-age suite and separates cleanly
from GRU, LSTM, and Mamba-lite writers under a shared training and evaluation
contract.  The result is ready to support an ICLR-style controlled-age memory
paper when the claim remains scoped to structured retention and exposure.

\section{Limitations}

The benchmark is controlled by design.  The cues are synthetic overlays on
rendered OGBench streams, which enables shortcut audits but does not cover all
real robot memory patterns.  The native-use claim depends on fixed controllers
and is reported only for environments with audited local controllers.  The
Mamba-lite baseline tests selective recurrence but is not the optimized
official Mamba stack.  The readout uses labels after training, so this is an
audit of exposed representation quality rather than an end-to-end policy.  Most
importantly, automatic age discovery remains future work: the current paper
tests registered delays, while the next stage must learn a delay-robust router
and report unseen-age generalization.
