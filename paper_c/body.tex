\section{Introduction}

Physical agents often observe the decisive evidence before the moment when an
action is chosen.  A robot may see an object label before it moves behind an
occluder, observe a door state before turning away, or pass a navigation cue
long before it reaches the branch where the cue matters.  Finite-context latent
world models make these settings tractable by compressing visual streams into a
short horizon, but that compression creates a specific failure mode: the model
may produce locally plausible futures while erasing the old evidence needed by
the downstream decision.  This paper studies that failure mode as an interface
problem rather than as a generic recurrent-memory benchmark.

The central question is not whether a side state exists.  A recurrent hidden
state can store information that the frozen host never exposes; a high-capacity
cache can memorize frames without learning which evidence will be needed; a
readout can decode a symbol without showing that the decoded symbol changes an
environment outcome.  We therefore use a stricter claim ladder.  First, the
task must require evidence outside the legal context window.  Second, the
memory state must retain that old evidence after context expiry.  Third, the
frozen host interface must expose it to a later readout.  Fourth, where an
audited controller exists, the memory-selected target should change native
environment success.

We instantiate this ladder with a masked-evidence JEPA objective and a compact
slot-memory writer.  The model is intentionally not given semantic labels,
cue crops, or hand-selected key frames during training.  Instead, salient
target frames and patch sets are mined from the rendered stream using generic
image statistics.  Those patches are withheld from the legal context and
predicted from a causal memory state.  Cue labels enter only after training,
when the audit asks whether the learned memory exposes the old cue under
full/reset/no-state contrasts.

The main empirical result is that the slot structure matters.  Across twelve
OGBench decks, three controlled evidence ages (4, 8, and 15), and three seeds,
the slot-memory model passes all 108 seed-level retention cells.  Under the
same caches, target mining, patch target encoder, loss, and post-hoc readout,
GRU, LSTM, and Mamba-lite baselines pass 73/108, 26/108, and 71/108 seed-level
cells.  The separation is largest at age 15: the slot-memory model reaches
0.962 mean full-memory balanced accuracy, while GRU, LSTM, and Mamba-lite reach
0.669, 0.350, and 0.683.  These are controlled-age stress tests.  They do not
yet claim automatic age discovery, but they establish that structured
slot-memory retention is not explained by a generic recurrent state.

\paragraph{Contributions.}
This paper makes four contributions.  First, it formulates old-evidence memory
as a frozen-interface audit with explicit demand, retention, exposure, and
bounded use gates.  Second, it introduces a non-manual masked-evidence
slot-memory objective that predicts latent patch sets rather than supervised
cue labels.  Third, it provides a controlled architecture comparison against
GRU, LSTM, and Mamba-lite across 12 OGBench decks, three ages, and three seeds,
with all non-memory components held fixed.  Fourth, it connects readout success
to fixed-controller native use on supported rows, reaching 0.957 mean executed
success over 27/36 scored environment-age rows while keeping AntMaze and
HumanoidMaze native-use limitations explicit.

\section{Problem setup and audit logic}

Let $x_{0:T}$ be a rendered observation stream and $a_{0:T-1}$ be the action
stream.  A cue $c$ appears early, before a decision time $t$, and determines a
target label $y$.  The frozen host observes only a finite recent context at
time $t$; the cue has already left that window.  A memory writer observes the
causal stream and produces state $M_t$.  A host-facing predictor exposes a
representation $h_t$ from the recent context and memory.  The audit tests
whether $h_t$ carries $y$.

The full condition uses the causal memory state.  The reset condition observes
the same endpoint and recent frames but clears memory at the context boundary.
The no-state condition removes persistent memory.  A seed-level cell passes
only if the full condition reaches balanced accuracy at least 0.750 while both
controls stay at most 0.350.  The threshold is fail-closed: the important
signal is not merely a high full readout, but a high full readout with matched
controls near chance.  This rules out endpoint color, action-only, and
short-context shortcuts.

The controlled ages 4, 8, and 15 are not arbitrary hyperparameters for the
method.  They are a measurement device.  Age 4 checks the near boundary after
context expiry; age 8 checks medium delay; age 15 is the long-memory stress
test within the current cache length.  Reporting all three prevents a method
from being selected only on its best temporal regime.  It also lets us separate
two failure modes: gradual age decay, which suggests storage or exposure
erosion, and immediate failure on object-heavy rows, which suggests that the
memory target is not rich enough for multi-object configurations.

\section{Masked-evidence slot transformer}

Figure~\ref{fig:architecture} shows the architecture in a Transformer-style
physical-intelligence view.  The input side contains visual patch tokens,
action tokens, and time tokens.  A frozen encoder maps rendered frames to token
features.  A compact slot-transformer writer updates eight memory slots through
attention and residual MLP blocks.  The target side mines salient patch sets
from the causal stream, masks those patches from the visible source, and
predicts stop-gradient color/texture latents.  The model therefore learns from
physical evidence that disappears from the legal context, not from a declared
class label.

\paragraph{Target mining.}
For each episode, temporal bins span the legal causal history up to the
endpoint.  A target frame is selected from each bin using generic visual
saliency and novelty.  Within each target frame, high-saliency patches are
mined with no access to cue labels, cue positions, or a task-specific crop.  The
selected target patches are removed from the visible stream.  A fixed patch
target encoder computes latent descriptors from average-pooled color, centered
color, and saturation statistics.  This target is deliberately generic.  It
pressures the memory to carry old visual evidence without telling the model
which semantic class will later be scored.

\paragraph{Slot memory and set prediction.}
The memory state is a small set of normalized slots.  Each slot receives
frame/action/time evidence through attention-style updates, and an MLP maps the
resulting slots to predicted target-patch latents.  The set loss contains a
set-level InfoNCE term, a cosine matching term, and a standard-deviation
regularizer to avoid collapse.  Since the target is a set, the model is not
rewarded for placing a specific patch in a hand-indexed slot.  It must instead
keep a structured summary that can expose whatever old evidence the audit later
queries.

\paragraph{Why compare to GRU, LSTM, and Mamba-lite?}
The baselines test whether the result comes from the objective alone or from
the structured memory interface.  GRU and LSTM replace the slot writer with a
single recurrent hidden state; the last valid hidden state is projected into
the same eight-slot predictor shape.  Mamba-lite is a plain PyTorch selective
diagonal state-space recurrence with input-dependent update gates and decay.  It
is not the official optimized Mamba implementation, but it tests the core idea
that selective recurrence might be sufficient.  All baselines share the same
data, target mining, patch encoder, loss, optimizer, and readout.

\section{Experimental protocol}

We evaluate twelve OGBench render decks: PointMaze-medium, PointMaze-large,
PointMaze-giant, PointMaze-teleport, AntMaze-large, AntMaze-giant,
HumanoidMaze-large, Cube-single, Cube-double, Cube-triple, Puzzle-3x3, and
Scene.  Each method runs at ages 4, 8, and 15 with seeds 0, 1, and 2, giving
108 seed-level cells per method.  The readout is fitted only after training.
The training loss never sees the cue labels.  We report mean full-memory
balanced accuracy, reset and no-state controls, seed-pass count, and the number
of environment-age rows where all three seeds pass.

The environment families are intentionally varied.  PointMaze rows test
spatial cue retention.  Teleport adds a discontinuous transition that can break
naive controller assumptions.  AntMaze and HumanoidMaze supply locomotion
visuals; for native use they remain controller-limited, but for retention they
are valid render-stage memory decks.  Cube rows test object identity and
multi-object ordering.  Puzzle and Scene are the hardest rows because the
relevant evidence is distributed over combinatorial or multi-object visual
state rather than a single colored waypoint.

\paragraph{Controls remain informative.}
The reset and no-state columns are not decorative.  In the main comparison,
all methods keep controls near chance on average.  That means GRU and
Mamba-lite failures are not caused by leakage in the controls, and our positive
rows are not explained by endpoint shortcuts.  The controls also explain why
high retrieval alone is insufficient: a target-patch retrieval score can be
high while the semantic readout is weak, especially on Puzzle and Scene.  The
paper therefore emphasizes full/reset/no-state exposure rather than only the
self-supervised training loss.

\section{Native-use checks}

Readout accuracy is not execution.  We therefore run a fixed-controller
native-use stage where a local audited controller exists.  The memory readout
selects a target; the controller receives that selected target, not the true
label; the environment then reports success or failure.  This is a bounded use
check, not native world-model planning.

The repaired native-use summary scores 27/36 environment-age rows with 0.957
mean full-memory executed success.  Recent-only and random target selection are
both near 0.279.  PointMaze-teleport is included after adding a teleport-aware
PointMaze controller gate; older controller attempts failed because the
stochastic teleport transition could send the agent into an isolated outlet.
Cube, Puzzle, and Scene rows use simple fixed controller compositions.  This
stage gives the paper a final-performance signal: exposed old evidence changes
the selected target, and the selected target changes native success.

The boundary is equally important.  AntMaze and HumanoidMaze have valid
render-stage retention results, but this repository has no audited low-level
expert controller for native execution.  Reporting them as native successes
would be misleading.  We therefore treat those rows as controller-unavailable,
not as method failures or successes.  This conservative accounting is the same
reason the paper separates retention, exposure, and use rather than collapsing
everything into one score.

\section{Insights and failure modes}

\paragraph{Structured slots help when evidence is distributed.}
The main insight is not simply that our method has more parameters.  The
comparison keeps dimension, target encoder, loss, and readout fixed.  The
slot interface changes how information is stored and exposed: multiple slots
can preserve distinct pieces of old visual evidence and present them as a set
to the patch predictor.  This matches the target.  A patch-set objective asks
for several visual facts; a slot set is a natural carrier for those facts.
Single-state recurrent baselines can solve simple waypoint-like rows but lose
more information when object identity, ordering, or scene configuration matters.

\paragraph{Host exposure is the right object of measurement.}
The dashboard evidence that led to this paper included earlier feature-host
and fixed-controller stages.  Those stages showed that old evidence can be
available in a carrier while still failing to become a useful host output.  The
current paper therefore measures exposed memory with post-hoc readouts and
native-use checks.  This is stricter than measuring internal reconstruction
loss, and it explains why we keep reset and no-state controls in every table.

\paragraph{Age is still controlled, not automatic.}
The current result should be written as a controlled-age memory paper.  Ages 4,
8, and 15 are manually specified stress tests.  We have started a separate
mixed-age training run, but the present paper does not claim that the model
automatically chooses how far back to search.  A future auto-age version should
add timestamped slots and a query router that scores slots by content match,
age, and confidence, then report unseen-age generalization and maximum
supported age.  That is an extension, not a premise of the current evidence.

\section{Limitations}

The benchmark is controlled by design.  The cue decks are synthetic overlays on
rendered OGBench streams, which makes shortcut auditing possible but does not
cover every real-world memory pattern.  The native-use stage depends on fixed
controllers, and only rows with audited controllers are scored.  Mamba-lite is
a useful selective state-space baseline, but it is not a replacement for an
optimized full Mamba implementation.  Finally, the readout uses cue labels
after training.  This is appropriate for an audit, but it is not itself an
end-to-end policy.  The claim is therefore precise: non-manual slot-memory
training improves old-evidence retention and host exposure under controlled
ages, and fixed-controller checks show that this exposed memory can change
outcomes where controller support exists.

\section{Conclusion}

Old-evidence memory in world-model interfaces should be evaluated by what is
retained, what is exposed, and what can be used.  Under that audit, a
masked-evidence slot transformer cleanly separates from GRU, LSTM, and
Mamba-lite baselines across twelve OGBench decks and three evidence ages.  The
result is strong enough for an ICLR-style paper when the scope is explicit:
structured slot memory solves controlled-age retention and exposure, and
fixed-controller use verifies that the exposed target can change native
success.  Automatic age discovery and native locomotion execution remain the
next experimental stages.
